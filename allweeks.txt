Creating and attaching an EBS volume to an EC2 instance
Create a New EBS Volume
Open the EC2 Dashboard.
From the left-hand menu, select Volumes under Elastic Block Store (EBS).
Volume type (gp3/gp2), Size of the volume , Availability Zone (must be the same as the EC2 instance)
Click Create Volume.

Attach the EBS Volume to the Existing EC2 Instance
Select the newly created volume.
Click Actions → Attach Volume.
Select the target EC2 instance from the list.
Specify the device name.
Click Attach.
Verify the Volume on the EC2 Instance
Connect to the EC2 instance using SSH.

Attached EBS volume and use it to create and attach a new volume to an EC2 instance in same AWS region.
Copy the ssh key of the instance A
Open the powershell navigate to the pem file directory , paste the ssh key

Check all available disks : lsblk
Check filesystem : lsblk -fs

Create a partition on the new volume : sudo fdisk /dev/nvme1n1
Press m → Help menu
Press n → New partition
Press w → Write changes and exit.
Notify kernel : partprob
Verify the New Partition : lsblk -fs

Format with XFS filesystem : sudo mkfs.xfs /dev/nvme1n1p1
Verify : lsblk –fs
Create a Mount Directory : sudo mkdir /mnt/ebstry
Mount the Volume : sudo mount /dev/nvme1n1p1 /mnt/ebstry
Verify : df –h
Make the Mount Persistent
Get the UUID of the volume : sudo blkid /dev/nvme1n1p1
Copy UUID.

Edit fstab: sudo nano /etc/fstab
Add UUID line at the end of the file.
UUID= d1f88f9b-c2b7-4973-b9cc-f2f4e377d62e /mnt/ebstry xfs defaults,nofail 0 0
Save and exit.
Test fstab: sudo systemctl daemon-reload
Sudo mount -a
Verify : df -h

Unmount volume from Instance-A : sudo umount /mnt/data
Check : df –h
Verify Persistence :
Reboot the instance : select the instance and click on reboot instance

Create the new instance B in the same AWS region

Detach Volume:
Select the volume and deattach with the instance A. Attach the volume to the instance B.

After attaching the volume to the new instance
Copy the ssh key of instance B and paste it in powershell

Create mount directory : sudo mkdir /mnt/ebstry
Mount the volume : sudo mount /dev/nvme1n1p1 /mnt/ebstry
All previous data will be available.

List the blocks to verify : lsblk

Create a snapshot of the attached EBS volume and use it to create and attach a new volume to an EC2 instance in another AWS region.
Deattach the volume to the instance in N.Virginia

Create a snapshot to the volume which is in N.Virginia .
Select the snapshot and click on Actions to select copy snapshot.
Copy the snapshot to another region ( Oregon ) .

Switch to the Destination Region.
Create a Volume from the Snapshot.
Create a new ec2 instance in the destination region & attach the Volume to the Instance.

After creating the instance attach the volume to the instance .

Select the instance and copy the ssh key.
Paste the ssh key in the powershell where the pem file exists.

Verify the Attached Disk : lsblk
Create a Mount Directory : sudo mkdir /mnt/ebstry
Mount the Volume : sudo mount /dev/nvme1n1p1 /mnt/ebstry
Verify : df -h

VPC Creation and Ec2 Instance Connection

Go to AWS Management Console → VPC

Click Create VPC -> name : my-vpc
IPv4 CIDR block: e.g. 10.0.0.0/16 -> Click Create VPC

Create subnets in same Availability Zones.
Select vpc -> provide the name as public-subnet
Choose AZ -> Enter CIDR ( 10.0.1.0/24 ) -> Create subnet

Select vpc -> provide the name as private-subnet
Choose AZ -> Enter CIDR ( 10.0.2.0/24 ) -> Create subnet

Go to Internet Gateways -> Click Create internet gateway -> Name : internet gateway
-> Attach it to your VPC

Go to Route Tables → Create route table
Select VPC -> name : public route

Create route table for private subnet -> Select VPC -> name : private-route

Select public route -> click on actions -> go to edit routes
Click on add route -> Destination: 0.0.0.0/0 -> Target: Internet Gateway
Select the gateway which was created -> save changes

Go to subnets -> Select Public Subnet -> Go to Edit subnet settings -> Enable Auto-assign public IPv4 address

Select public route -> click on edit subnet associations
Select public subnet -> save associations

Similarly do for private subnet

Create a Security Group -> Add inbound rules:
o HTTP (80) / HTTPS (443)
o SSH (22)

Go to EC2

Create a instance -> select Ubuntu as AMI
Edit the network settings -> select vpc -> select public subnet -> enable public ip -> launch the instance

Create another instance -> select Ubuntu as AMI
Edit the network settings -> select vpc -> select private subnet -> disable the public ip -> launch instance

Select public instance -> click on connect

Select private instance -> click on connect
Private instance will not be connected
